// PayPal Business einrichten
1. Gehe zu paypal.com/de/business und erstelle ein kostenloses Business-Konto.
2. Verwende die dort hinterlegte E-Mail-Adresse im Feld oben.
3. PayPal berechnet ca. 1,9% + 0,35€ pro Transaktion (nur bei Zahlung).
4. Du erhältst eine E-Mail-Benachrichtigung bei jeder Zahlung.

E-MAIL EINRICHTUNG

Wenn du einen Termin bestätigst, erhält der Kunde automatisch eine E-Mail mit dem PayPal-Zahlungslink. Dafür wird EmailJS genutzt — kostenlos bis 200 Mails/Monat.

Einrichtung in 3 Schritten:

1. Gehe zu emailjs.com → kostenloses Konto erstellen
2. Unter Email Services deinen Gmail- oder Outlook-Account verbinden → SERVICE_ID notieren
3. Unter Email Templates eine neue Vorlage erstellen mit diesen Variablen:

  Subject: Termin bestätigt – {{service}}
  Body: 
Hallo {{to_name}},

dein Termin wurde bestätigt! 🎉

📅 Datum:     {{date}}
⏰ Uhrzeit:   {{time}} Uhr
🏃 Leistung:  {{service}}

Bitte bezahle jetzt über folgenden Link:
{{paypal_url}}

Bei Fragen antworte einfach auf diese E-Mail.

Viele Grüße,
Matze
  → TEMPLATE_ID notieren
4. Unter Account den PUBLIC_KEY notieren
5. Alle drei Werte ganz oben in der admin.html eintragen

// Erste Schritte
SUPABASE EINRICHTUNG
1
Supabase-Projekt erstellen
Gehe zu supabase.com, erstelle ein kostenloses Konto und klicke auf New Project. Notiere dir danach unter Project Settings → API die zwei Werte: Project URL und anon public key.
2
Datenbank-Tabellen anlegen
Gehe im Supabase-Dashboard zu SQL Editor und führe dieses SQL aus:

-- Buchungen
create table bookings (
  id bigint generated always as identity primary key,
  fname text, lname text, email text, phone text,
  service text, duration text, date text, time text,
  message text, paid numeric default 0,
  status text default 'neu',
  created_at timestamptz default now()
);

-- Einstellungen (Verfügbarkeit, PayPal)
create table settings (
  key text primary key,
  value jsonb
);
3
Row Level Security (RLS) konfigurieren
Im SQL Editor ausführen:

-- RLS aktivieren
alter table bookings enable row level security;
alter table settings enable row level security;

-- Kunden dürfen Buchungen eintragen
create policy "insert bookings" on bookings for insert to anon with check (true);

-- Kunden dürfen Einstellungen lesen (für Verfügbarkeit & PayPal)
create policy "read settings" on settings for select to anon using (true);

-- Nur Admin darf alles lesen/schreiben
create policy "admin all bookings" on bookings for all to authenticated using (true);
create policy "admin all settings" on settings for all to authenticated using (true);
4
Admin-Benutzer anlegen
Gehe zu Authentication → Users → Add User im Supabase-Dashboard. Trage dort deine Admin-E-Mail und ein sicheres Passwort ein. Mit diesen Zugangsdaten loggst du dich in dieses Admin-Panel ein.
5
URLs in beide HTML-Dateien eintragen
Trage deine SUPABASE_URL und SUPABASE_KEY in beide Dateien ein — in homepage_secure.html ganz oben im Script-Block, und in admin.html direkt darunter.
⚠ Wichtig: Der anon public key ist für Kunden sichtbar — das ist normal und sicher, solange RLS korrekt konfiguriert ist (Schritt 3). Er erlaubt nur die freigegebenen Aktionen (Buchung eintragen, Einstellungen lesen). Der service_role Key hingegen darf niemals in den Frontend-Code.
